# ADR 0003 - Arquitetura de Pastas

## Status

Aceito.

## Contexto

Era necessario organizar projeto para escalar sem acoplamento.

## Decisao

Separar `app`, `features`, `components`, `lib`, `types`, `styles` e `docs`.

## Consequencias

- Facilita manutencao e onboarding.
- Exige disciplina para manter responsabilidades claras.
